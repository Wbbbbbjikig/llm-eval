# LLM 输出质量评测 Pipeline（信用风险三分类）

一个面向 **LLM 输出质量评测**的完整 pipeline：以信用风险三分类为任务，构建带金标准的
评测集，对多个 Prompt 变体做 A/B 评测，并对 badcase 做系统性归因，形成
「评测—归因—迭代」数据飞轮。

## 核心结果（本地规则模拟引擎实测）

| Prompt 变体 | 准确率 | 格式合规率 | 幻觉率 | 宏平均 F1 |
|---|---|---|---|---|
| v1 零样本（基线） | 70.0% | 87.5% | 0.0% | 0.75 |
| v2 角色+Rubric | **87.5%** | 100.0% | 0.0% | 0.88 |
| v3 Few-shot | 87.5% | 100.0% | 0.0% | 0.88 |
| v4 CoT | 87.5% | 100.0% | 0.0% | 0.88 |
| v5 组合策略 | 87.5% | 100.0% | 0.0% | 0.88 |

- **准确率由零样本基线 70.0% 提升至 87.5%（+17.5pct）**
- **格式合规率由 87.5% 提升至 100%**
- **badcase 归因**：错误集中在「中风险 ↔ 低风险」边界（中→低 2 例、低→中 2 例、高→中 1 例），
  定位边界模糊为主因，反哺 Prompt 的分档 Rubric 设计。

## 评测维度

| 维度 | 说明 |
|---|---|
| **准确率 Accuracy** | 预测风险等级与金标准一致的比例 |
| **格式合规率** | 输出是否为规定的 JSON 结构（可自动解析） |
| **幻觉/越界率** | 输出的风险等级是否超出三分类合法标签 |
| **宏平均 F1** | 三个类别 F1 的平均，抵消类别不平衡影响 |

## 项目结构

```
llm-eval/
├── data/eval_set.jsonl    # 40 条信用风险评测集（含金标准答案）
├── prompts/variants.py    # 5 个 Prompt 变体（零样本/Rubric/Few-shot/CoT/组合）
├── src/mock_llm.py        # 规则模拟层：无 API key 时打通全流程

```

## 运行方法

### 方式一：零依赖跑通（规则模拟，推荐先试）

```bash
cd llm-eval
python3 src/evaluate.py
```

无需 API key，用规则模拟层即可复现上面的全部结果与报告。

### 方式二：接入真实 LLM（OpenAI 兼容接口，含国内模型）

```bash
pip install openai
export OPENAI_API_KEY="你的key"
# 用国内模型（如通义千问）再设：
# export OPENAI_BASE_URL="https://dashscope.aliyuncs.com/compatible-mode/v1"
# export EVAL_MODEL="qwen-plus"
python3 src/evaluate.py
```

代码会自动检测 `OPENAI_API_KEY`，有则走真实 API、无则走规则模拟，其余流程完全一致。

## 方法论：评测—归因—迭代 数据飞轮

1. **Benchmark 搭建**：定义任务、构建含金标准的评测集、设计可自动打分的多维指标；
2. **A/B 评测**：系统对比多个 Prompt 变体，量化每个策略的效果增量；
3. **badcase 归因**：按错误类型（判断错误/格式错误/类别幻觉）与混淆方向拆解失败样本，
   定位系统性问题；
4. **反哺迭代**：将归因结论转化为 Prompt 设计改进（如针对边界模糊增加分档 Rubric），
   进入下一轮评测，形成闭环。

## 说明

评测引擎默认为规则模拟层，用于离线、零成本地打通与验证整套评测框架；接入真实 LLM
只需配置环境变量，评测集、打分逻辑、归因流程均可直接复用。项目重点在于**可复现的评测
标准、多维打分、A/B 对比与 badcase 系统性归因**这套方法论。
